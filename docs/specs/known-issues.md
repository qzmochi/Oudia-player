# 既知の課題・改善予定

## 方向ラベル問題

OuDia の Kudari/Nobori は駅リスト順で機械的に決まるため、実際の路線の上り/下りと一致しない場合がある。

例: 京成線サンプル
- ファイルの "Kudari" = 成田空港→上野 = 実際は上り
- ファイルの "Nobori" = 上野→成田空港 = 実際は下り

線路配置（左側通行に基づく上下）は物理方向ベースなので視覚的には正しい。
情報パネルの方向ラベルが実際と逆になる問題あり。

→ 対応案: ユーザーが方向ラベルを反転できる設定を追加

## 分岐駅の表示

京成線の駒井野のように、分岐点の駅が駅リストに2回出現するケースがある。
現在は2つの独立した駅として扱っているため、CTC表示上は分岐として表現されていない。

→ 対応案: BrunchCoreEkiIndex を使って分岐構造を検出し、Y方向に分岐描画

## 駅配線・ホーム表現

現在は駅マーカーのみで、番線ごとのホーム表現がない。
EkiTrack2 のデータ（番線数、DownMain/UpMain）と RessyaTrack（列車の使用番線）を使って、
列車がどのホームに入線するかを表現したい。

→ 対応案:
1. 駅構内を拡大表示（駅ゾーン内に番線を描画）
2. 列車が駅に近づくと番線方向に分岐するアニメーション
3. 退避（待避線に入る列車）の表現

## 連結・解結

OuDia に直接のフィールドがない。Bikou（備考）からの推定 or 独自定義が必要。

## 単線/複線

駅間の線路本数は OuDia に含まれない。独自補足データで対応が必要。
